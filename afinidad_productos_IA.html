Este formato es el que existe en cualquier SGA o ERP. Un pedido multilinea ocupa varias filas — una por producto. El problema es que así los datos no dicen nada sobre qué productos van juntos. Eso lo calcula el paso siguiente.
¿Qué hace esta consulta?
El LLM no puede leer 4.840 filas de pedidos. Esta consulta SQL las convierte en una tabla compacta de co-ocurrencias: cuántas veces aparecen juntos dos productos en el mismo pedido. Eso sí lo puede analizar el LLM.
Consulta SQL — paso 1: normalizar productos por pedido
-- Primero: una fila por par de productos dentro del mismo pedido SELECT a.pedido_id, a.producto AS producto_a, b.producto AS producto_b, a.fecha FROM pedidos a JOIN pedidos b ON a.pedido_id = b.pedido_id AND a.producto < b.producto -- evita duplicados (A,B) y (B,A) WHERE a.fecha >= CURRENT_DATE - INTERVAL '90 days'
Consulta SQL — paso 2: agregar y calcular afinidad
WITH pares AS ( -- subconsulta del paso 1 arriba ... ), totales AS ( -- cuántos pedidos tiene cada producto en total SELECT producto, COUNT(DISTINCT pedido_id) AS total_pedidos FROM pedidos WHERE fecha >= CURRENT_DATE - INTERVAL '90 days' GROUP BY producto ) SELECT p.producto_a, p.producto_b, COUNT(*) AS pedidos_juntos, t.total_pedidos AS total_a, ROUND(COUNT(*) * 100.0 / t.total_pedidos, 1) AS pct_afinidad FROM pares p JOIN totales t ON t.producto = p.producto_a GROUP BY p.producto_a, p.producto_b, t.total_pedidos HAVING COUNT(*) >= 50 -- descarta pares anecdóticos ORDER BY pct_afinidad DESC LIMIT 20;

De 4.840 filas a 10 filas. El LLM recibe un resumen que cabe en el prompt, no la base de datos entera. Esta separación de tareas — SQL para agregar, LLM para razonar — es clave en cualquier sistema de IA logística.

¿Qué hace el sistema después?
  • 1
    El JSON se convierte en órdenes de cambio de ubicación en el SGA
    Sin intervención manual — el sistema genera las instrucciones de movimiento automáticamente
  • 2
    El responsable revisa y aprueba antes de ejecutar
    El LLM propone, la persona decide. Nada se mueve sin validación humana
  • 3
    El ciclo se repite cada semana o mes
    En diciembre "repostería" crece, en verano "bebidas" se separa. Los grupos evolucionan con la demanda
Cómo enseñarlo en clase

Empieza por el problema, no por la solución

Pregunta: "¿Cómo sabéis ahora qué productos ponen juntos los clientes?" Deja que respondan. El silencio incómodo es el gancho.

El paso SQL es el más importante

La IA no hace magia con datos caóticos. Muestra la pestaña 2 primero: alguien tiene que limpiar y agregar los datos antes de que el LLM pueda razonar.

Las restricciones son la clave

Pide a los alumnos que añadan una restricción de su propio almacén al prompt. Eso convierte el ejemplo genérico en algo real para ellos.

El JSON como punto de debate

Muestra la respuesta del LLM y pregunta: "¿Aprobaríais este resultado tal cual?" Busca que discutan qué cambiarían antes de enviarlo al SGA.

La pregunta que más debate genera

"¿Cada cuánto tiempo deberíamos recalcular los grupos?" No tiene respuesta única — depende de estacionalidad, campañas, altas de producto.

Conecta con el slotting dinámico

Si ya vieron el ejemplo anterior, pregunta: "¿Cuál haríais primero, el slotting o la afinidad?" La respuesta correcta es que se alimentan mutuamente.

El mensaje central para llevar a casa: el LLM no sustituye el criterio del gestor ni el sistema de datos. Hace una sola cosa bien — razonar sobre restricciones contradictorias y explicar por qué — que un SQL no puede hacer.
